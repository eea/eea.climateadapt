GeoNetwork - Prise en main

Lancement de l'application

Pour utiliser GeoNetwork, vous devez d'abord démarrer le serveur Web intégré.

Sous Windows, accédez à GeoNetwork dans le menu windows qui a été créé lors de l'installation et lancer "Démarrer serveur". Vous pouvez ensuite ouvrir l'application dans votre navigateur.

Par défaut, ce devrait être au http://localhost:8080/geonetwork

IMPORTANT!!Après avoir ouvert la page d'accueil de GeoNetwork, connectez-vous en tant qu'administrateur. Vous devez utiliser le nom d'utilisateur admin et le mot de passe admin. Dans l'Administration, vous pouvez ensuite modifier votre compte.

Documentation en ligne est disponible sur le GeoNetwork page d'accueil. Sélectionnez Aide de la barre de menu. Un lien vers la version imprimable version PDF peut être trouvé au bas de la page d'aide.

Vous pouvez également démarrer le serveur Web Jetty de GeoNetwork utilisant les scripts situés dans le $INSTALL_PATH/bin directory.

Sur une plate-forme Windows, dans le répertoire bin de votre installation :

  • start-geonetwork.bat
  • stop-geonetwork.bat
Sous Linux ou mac OSX, dans le répertoire bin de votre installation :
  • start-geonetwork.sh
  • stop-geonetwork.sh

Note: Un environnement d'exécution Java (JRE) @javaVersion@ Standard Edition (téléchargement depuis http://java.sun.com) est requis pour l'exécution GeoNetwork opensource. Voir les sections Dépannage normale si l'installation ou le démarrage échoue.

Chargement des métadonnées

Le catalogue est fourni avec des modèles de métadonnées par défaut pour vous permettre de créer de nouvelles métadonnées en utilisant l'éditeur en ligne.

Si vous voulez charger d'autre métadonnée, plusieurs options s'offrent à vous:

  • Import de fichier XML via la section administration du catalogue
  • Import de fichier MEF via GAST ou la section administration du catalogue
  • Moissonage

Le format MEF a été spécialement conçus pour l'échange de métadonnées et de données, les aperçus ainsi que les privilèges d'accès des catalogues de type GeoNetwork. Vous pouvez trouver un échantillon de fichiers MEF dans le \gast\setup\sample-data dossier. Le format de fichier a été décrit dans le chapitre 3 du serveur-manuel de référence (dans \docs\manuals\. Ce format sera également disponible dans d'autres applications géospatiales.
Pour importer ou exporter des métadonnées de GeoNetwork, vous devez démarrer l'outil GeoNetwork Administrateur Survival Tool (GAST).

  • Tout d'abord configurer votre connexion GeoNetwork serveur en fournissant des détails de compte et dans le menu Options => Config menu. Note: Vous avez besoin d'un éditeur ou un administrateur compte pour que cette fonction de travail.
  • Pour charger échantillon de métadonnées et de données, sélectionnez l'option échantillon de données dans la base de données et appuyez sur la section d'importation
  • Pour charger d'autres données mises en forme MEF, sélectionnez Importer dans la gestion des options. Sélectionnez le dossier contenant les fichiers MEFet cliquez sur le bouton Importer.

Configuration du système

Serveur Web

L'application est embarquée avec une Jetty serveur Web. Vous pouvez utiliser Jakarta Tomcat version 5.5 au lieu si nécessaire. Vous avez à mettre en place de nouveaux contextes dans Tomcat qui pointent vers les applications web dans le dossier de l'installation GeoNetwork.

System de base de données

GeoNetwork de bureau est livré avec un serveur de bases de données internes, la base de données SQL McKoi.

Si vous prévoyez d'utiliser un autre SGBD, vous pouvez utiliser GAST à mettre en place la base de données et migrer le contenu de la base de données embarquée McKoi. Assurez-vous que le SGBD est déjà en place et fonctionne avec un schéma de base de données et un utilisateur. Sélectionnez le SGBD en utilisant les GAST outil. Oracle, MySQL et PostgreSQL sont actuellement supportés. GeoNetwork utilise l'interface JDBC de travailler avec le SGBD, pour que d'autres SGBD systèmes de mai également.

Vous pouvez également configurer manuellement le SGBD, bien que cela soit plus complexe (voir le site Web de la communauté pour plus de détails)

Pour plus d'informations, à http://geonetwork-opensource.org
